# Containers Fundamentals training material

Modern software development requires applications to run reliably across different environments such as developer laptops, testing servers, and cloud platforms. One of the biggest challenges in software deployment is ensuring that applications behave consistently regardless of where they run. Containers solve this problem by packaging applications together with their dependencies into a single, portable unit.

A container is a lightweight and isolated environment that includes everything needed to run an application. This typically includes the application code, runtime, system tools, libraries, and configuration files. Because all dependencies are packaged together, the application can run consistently on any system that supports container technology.

Unlike traditional virtual machines, containers do not require a full operating system for each application. Instead, containers share the host operating system kernel while keeping applications isolated from each other. This makes containers much more efficient in terms of performance, startup time, and resource usage.

Containers are widely used in modern software development practices such as microservices architecture, DevOps, and cloud-native applications. They allow developers to build, test, and deploy applications quickly and reliably.

In this hands-on workshop the attendee learns the fundamental skills needed to understand the concepts and start to choose a container runtime, setup a container runtime like Docker Desktop, develop container images and deploy container based solutions, which share virtual network or storage resources.
